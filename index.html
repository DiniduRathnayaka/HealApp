Research Problem & Solution

Research Problem

  • How elders track their daily health with expensive health trackers and their complicated user interfaces?
  • How elders travel to pharmacies to purchase drugs with a risk to travel with their age and pandemic situations?
  • From whom will the elders get health tips related to their current medical situation?
  • In emergency from whom they get help to contact emergency units and predict their diseases?

Proposed Solution

When user register on the system it activates the device and obtains the current body factor as an initial calibration and store the data in a Realtime database. Normally the readings will be updated to the database on a 30 second timeframe. When the gadget is activated, the user may also have the access to manually measure these elements at any time through the mobile application. At the end of the month, the user can get fully detailed charts on his/her health factors variation. This chart includes heart rate/blood pressure, body temperature and blood oxygen level. Also, by using the data a trained machine learning algorithm is used to give future predictions which are diseases and health problems the user may experience in the future. These predictions are given by analyzing the sensory data which are stored in the Realtime database. The elderly, as well as other patients, have had difficulties with medicine usage for a long time. The prescriptions written by doctors are scanned using optical character recognition (OCR) and then sent to the pharmacy. The pharmacy will receive a pdf of the dataset, which they may access using their pharmacy software related to the device. If the medications are in stock, the pharmacy provides the go-ahead and sends them out via a delivery service; otherwise, the pharmacy declines the request, and the system locates the next closest pharmacy site for the patient to pick up their medication. An algorithm developed to evaluate a user's present medical state is used to provide a variety of workouts, diet regimens as health tips. METHODOLOGY

The heart rate, blood pressure, blood oxygen level, and body temperature may all be measured using Internet of Things (IoT) device in this system. Sensors will record these elements as user inputs, which will be kept in a Realtime database for other functions in the mobile application. The sensor readings are analyzed to give future health predictions for the user by a trained machine learning algorithm. Also, the user can see generated charts for his/her blood oxygen level, body temperature and heart rate. The same readings form the IoT device are analyzed to give health tips for the user. There is a trained dataset which will be used to give these predictions. An advanced chatbot is present in the application which can be used to predict heart related diseases. The chatbot will ask questions from the user and predict the disease and also it will suggest doctors to get medical assistance. It also has an emergency assistance feature where it can be used to send alerts to 1919 like emergency hotlines. The prescriptions given by the doctors can be read through the mobile application. These data can be sent to pharmacies near to the user’s location. The pharmacies are showed via the mobile application to the user by filtering them by their distance. The user can send the prescription to the nearest one. Providing health tips as exercises and diet plans is another major function of this application. These health tips are also given based on the users current readings.
